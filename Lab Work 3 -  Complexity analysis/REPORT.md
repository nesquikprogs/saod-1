## Тема

Анализ сложности алгоритмов.

## Задание

По списку чисел и среднему значению определить, каких элементов больше:
- больших среднего;
- меньших среднего.

Нужно описать группы входных данных, лучший/худший/средний случаи и провести тестирование с выводом в текстовый файл.

## Алгоритм

1. Проходим по списку один раз.
2. Считаем:
   - `greaterCount` — сколько элементов больше среднего;
   - `lessCount` — сколько элементов меньше среднего.
3. Элементы, равные среднему, не учитываются.
4. Используем раннюю остановку:
   - `remaining` — сколько элементов осталось;
   - `diff = greaterCount - lessCount`;
   - если `|diff| > remaining`, итог уже не изменится, можно закончить раньше.

## Группы входных данных

1. Элементов больше среднего больше (`greaterCount > lessCount`).
2. Элементов меньше среднего больше (`lessCount > greaterCount`).
3. Количество одинаково (`greaterCount = lessCount`).
4. Есть элементы, равные среднему (они не влияют на итог).

## Оценка сложности

Пусть `n` — размер списка.

- Лучший случай: `O(1)` (ответ понятен очень рано).
- Худший случай: `O(n)` (нужно просмотреть весь список).
- Средний случай: `O(n)`.

## Тестирование

В основной программе реализован запуск набора тестовых примеров.

Проверяются случаи:
- ранний выход;
- худший случай (ничья в конце);
- случай, когда меньше среднего больше;
- случай с элементами, равными среднему;
- все элементы больше среднего;
- все элементы меньше среднего.

После запуска программы результаты автоматически сохраняются в файл `test_results.txt`.

## Вывод

Реализован алгоритм сравнения количества элементов относительно среднего значения.  
Алгоритм корректен, работает за линейное время в худшем случае и поддерживает раннюю остановку.
