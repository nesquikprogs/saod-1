BUBBLE SORT (ПУЗЫРЬКОВАЯ)

ИДЕЯ:
На каждом проходе сравниваем соседние элементы и меняем местами, если
стоят не по порядку. Самый большой элемент "всплывает" в конец.

BIG O:
**Лучший случай (O(n))**: массив уже отсортирован по возрастанию, на первом проходе `swapped` остается `false`, алгоритм сразу завершается
**Средний случай (O(n^2))**: случайный массив, обычно нужно много обменов и почти все проходы
**Худший случай (O(n^2))**: массив отсортирован в обратном порядке, максимум обменов на каждом проходе
Сложность пузырьковой сортировки O(n²), потому что внешний цикл делает порядка n проходов, 
а внутренний в сумме выполняет (n−1) + (n−2) + ... + 1(последний член суммы) сравнений, что по порядку равно n²
Память: O(1)
```cpp
template <typename T>
void bubbleSort(T arr[], int n)
{
    // Внешний цикл = количество проходов по массиву, -1 тут потому, что последний проход уже бессмысленный
    for (int i = 0; i < n - 1; ++i)
    {
        bool swapped = false; // флаг, был ли обмен на этом проходе

        // Внутренний цикл сравнивает пары arr[j] и arr[j+1]
        // n-1-i потому что хвост уже отсортирован после предыдущих проходов(для уменьшения количества сравниавемых элементов)
        for (int j = 0; j < n - 1 - i; ++j)
        {
            if (arr[j] > arr[j + 1])
            {
                std::swap(arr[j], arr[j + 1]);
                swapped = true;
            }
        }

        // Если за проход обменов не было - массив уже отсортирован
        if (!swapped)
            break;
    }
}
```
INSERTION SORT (СОРТИРОВКА ВСТАВКАМИ)

ИДЕЯ:
Держим левую часть массива отсортированной. Берем очередной элемент key,
идем влево по уже отсортированной части, пока элементы больше key, сдвигаем их вправо, когда нашли место, вставляем key
сдвигая большие элементы вправо.

BIG O:
- **Лучший случай (O(n))**: массив уже отсортирован, внутренний `while` почти не выполняется
- **Средний случай (O(n^2))**: случайный массив, каждый элемент в среднем сдвигается на заметную часть пути влево
- **Худший случай (O(n^2))**: массив в обратном порядке, каждый новый элемент приходится сдвигать через всю отсортированную часть
Сложность сортировки вставками O(n^2) потому что для каждого элемента внешний цикл может сдвигать во внутреннем цикле почти всю уже отсортированную часть массива (1 + 2 + ... + (n-1))
- Память: O(1)
```cpp
template <typename T>
void insertionSort(T arr[], int n)
{
    // Начинаем со второго элемента, первый сам по себе уже "отсортирован"
    for (int i = 1; i < n; ++i)
    {
        T key = arr[i]; // элемент, который вставляем
        int j = i - 1; // идем влево по отсортированной части(индекс элемента слева)

        // Пока слева элементы больше key и не вышли за левую границу
        while (j >= 0 && arr[j] > key)
        {
            arr[j + 1] = arr[j]; // сдвигаем элементы вправо 
            --j; // идем дальше влево
        }

        // Вставляем key в освободившееся место(после элемента, если элемент слева не больше него/в начало, если все элементы слева были больше key)
        arr[j + 1] = key;
    }
}
```
MERGE SORT (СОРТИРОВКА СЛИЯНИЕМ)

ИДЕЯ:
"Разделяй и властвуй":
1) делим массив пополам до отрезков длины 1
2) рекурсивно сортируем половины
3) сливаем две отсортированные половины в один отсортированный массив

BIG O:
- **Лучший случай (O(n log n))**: любой вход
- **Средний случай (O(n log n))**: любой вход
- **Худший случай (O(n log n))**: любой вход
Сложность сортировки слиянием O(n log n), потому что массив рекурсивно делится пополам до глубины log n, и на каждом уровне рекурсии все операции слияния вместе обрабатывают суммарно n элементов.
- Память: O(n) + O(log n) стек рекурсии
```cpp
template <typename T>
void merge(T arr[], size_t left, size_t mid, size_t right)
{
    // Длины левого и правого подмассивов
    size_t n1 = mid - left + 1;
    size_t n2 = right - mid;

    // Временные массивы под копии
    T* L = new T[n1];
    T* R = new T[n2];

    // Копируем данные во временные массивы
    for (size_t i = 0; i < n1; ++i)
        L[i] = arr[left + i];
    for (size_t j = 0; j < n2; ++j)
        R[j] = arr[mid + 1 + j];

    // Индексы для слияния
    size_t i = 0; // текущая позиция в L
    size_t j = 0; // текущая позиция в R
    size_t k = left; // текущая позиция в исходном массиве arr

    // Пока в обоих подмассивах есть элементы — выбираем минимальный
    while (i < n1 && j < n2)
    {
        if (L[i] <= R[j]) // если меньше левый символ
        {
            arr[k] = L[i]; // пишем левый
            ++i;
        }
        else // если меньше правый символ
        {
            arr[k] = R[j]; // пишем правый
            ++j; 
        }
        ++k;
    }

    // Добрасываем хвост левого подмассива (если остался)
    while (i < n1)
    {
        arr[k] = L[i];
        ++i;
        ++k;
    }

    // Добрасываем хвост правого подмассива (если остался)
    while (j < n2)
    {
        arr[k] = R[j];
        ++j;
        ++k;
    }

    delete[] L;
    delete[] R;
}

template <typename T>
void mergeSort(T arr[], size_t left, size_t right)
{
    // База рекурсии: 0 или 1 элемент — уже отсортировано
    if (left >= right)
        return;

    size_t mid = left + (right - left) / 2;

    // Рекурсивно сортируем левую и правую части
    mergeSort(arr, left, mid);
    mergeSort(arr, mid + 1, right);

    // Сливаем два отсортированных подмассива
    merge(arr, left, mid, right);
}
```
QUICK SORT (БЫСТРАЯ СОРТИРОВКА)

ИДЕЯ:
1) выбираем опорный элемент (pivot)
2) делаем partition: слева элементы < pivot, справа >= pivot
3) рекурсивно сортируем левую и правую часть

BIG O:
- **Лучший случай (O(n log n))**: pivot делит массив примерно пополам на каждом шаге
- **Средний случай (O(n log n))**: случайный массив, в среднем разбиения достаточно сбалансированные
- **Худший случай (O(n^2))**: каждый раз очень плохое разбиение (0 и n-1), часто бывает на уже отсортированном или обратно отсортированном массиве при фиксированном выборе pivot
Сложность quick, потому что каждый partition делает линейный проход O(n) по текущему подмассиву, и при сбалансированных разбиениях получается O(n log n), а при вырожденных разбиениях вида 0 и n-1 — O(n^2).
Память:         O(log n) стек (в среднем), O(n) в худшем
```cpp
template <typename T>
int partition(T arr[], int low, int high)
{
    T pivot = arr[high]; // опорный элемент
    int i = low - 1; // граница “зоны элементов меньше pivot” (изначально зона пустая)

    // По элементам массива
    for (int j = low; j < high; ++j)
    {
        if (arr[j] < pivot)
        {
            ++i; // расширяем зону меньших pivot
            std::swap(arr[i], arr[j]); // текущий элемент переезжает влево
        }
    }

    // Ставим опорный элемент сразу после элементов, меньших pivot
    std::swap(arr[i + 1], arr[high]);
    return i + 1;
}

template <typename T>
void quickSort(T arr[], int low, int high)
{
    // База рекурсии: пустой подмассив или один элемент
    if (low >= high)
        return;

    // Разбиваем массив и получаем позицию опорного элемента
    int pivotIndex = partition(arr, low, high);

    // Рекурсивно сортируем левую и правую части
    quickSort(arr, low, pivotIndex - 1);
    quickSort(arr, pivotIndex + 1, high);
}
```
SHELL SORT (СОРТИРОВКА ШЕЛЛА)

ИДЕЯ:
Это "ускоренные вставки": сначала сортируем элементы, стоящие далеко
друг от друга (шаг gap), затем уменьшаем gap и повторяем.
Когда gap становится 1, это обычные вставки, но массив уже "почти
отсортирован", поэтому работает быстрее на практике.

BIG O (зависит от последовательности gap):
- **Лучший случай**: близок к `O(n log n)` или лучше на почти отсортированных данных (зависит от последовательности `gap`)
O(n) — это чаще про обычные вставки; у Шелла из‑за нескольких gap лучший случай обычно описывают как «мало работы внутри фаз», но фазы всё равно остаются
- **Средний случай**: зависит от `gap`, обычно заметно лучше `O(n^2)` на практике
- **Худший случай**: для простого `gap = n/2, n/4, ...` обычно указывают до `O(n^2)`
- Важный момент: точная асимптотика сильно зависит от выбранной последовательности шагов
Шелл — это ускоренные вставки за счёт «грубой» перестановки на больших расстояниях; на практике часто заметно быстрее простых вставок на случайных данных
Сортировка Шелла такая по сложности, потому что на каждом из O(log n) значений gap вы делаете O(n) вставок со сдвигами, а в худшем случае для последовательности n/2, …, 1 эти сдвиги всё ещё могут суммарно дать O(n²) сравнений и перестановок, хотя на типичных данных большие gap сильно сокращают «длинные» протаскивания по сравнению с обычными вставками
Память: O(1)
```cpp
template <typename T>
void shellSort(T arr[], int n)
{
    // Проходы с уменьшающимся шагом: n/2, n/4, …, 1 (на 1 — как вставками)
    for (int gap = n / 2; gap > 0; gap /= 2)
    {
        // Вставка в отсортированную часть каждой подпоследовательности с шагом gap
        for (int i = gap; i < n; ++i)
        {
            T key = arr[i];
            int j = i - gap; // левый сосед в той же подпоследовательности с шагом gap
            // Пока слева элементы больше key и не вышли за левую границу
            while (j >= 0 && arr[j] > key)
            {
                arr[j + gap] = arr[j]; // сдвигаем элементы вправо 
                j -= gap; // идем дальше влево с шагом gap
            }
            // Вставляем key в освободившееся место(через gap после элемента, если элемент слева не больше него/в начало, если все элементы слева были больше key)
            arr[j + gap] = key;
        }
    }
}
```
SELECTION SORT (СОРТИРОВКА ВЫБОРОМ)

ИДЕЯ:
На каждом шаге находим минимум в неотсортированной части массива
и ставим его в начало этой части.

BIG O:
- **Лучший случай (O(n^2))**: любой вход
- **Средний случай (O(n^2))**: любой вход
- **Худший случай (O(n^2))**: любой вход
- Почему одинаково: на каждом шаге минимум в хвосте ищется полным просмотром, число сравнений почти не зависит от порядка элементов
Внешний цикл ~n раз, внутренний на каждом i до ~n итераций → порядка n² сравнений; количество обменов мало (не больше n-1). Память O(1).
Примечание: сравнений всегда примерно n^2/2, независимо от входа.
```cpp
template <typename T>
void selectionSort(T arr[], int n)
{
    for (int i = 0; i < n - 1; ++i)// n - 1: когда дошли до предпоследнего, последний уже окажется на своём месте автоматически.
    {
        int minIndex = i; // индекс минимального элемента

        // Ищем индекс минимального элемента
        for (int j = i + 1; j < n; ++j) // j - следующий элемент
        {
            if (arr[j] < arr[minIndex]) // если найденный элемент меньше минимального
                minIndex = j; // используем его как новое минимальное
        }

        // Ставим минимум на позицию i
        if (minIndex != i)
            std::swap(arr[i], arr[minIndex]);
    }
}
```
RADIX SORT (ПОРАЗРЯДНАЯ) - версия для int >= 0

ИДЕЯ:
Сортируем не сравнениями, а по разрядам:
единицы -> десятки -> сотни -> ...
Для каждого разряда используем стабильную counting sort.

BIG O:
- **Лучший/средний/худший**: обычно одинаковый порядок `O(d * (n + k))`, потому что алгоритм делает фиксированное число проходов по разрядам
- Где:
    - `d` — количество разрядов у максимального числа
    - `k` — размер алфавита разряда (для десятичной системы `k = 10`)
- Для фиксированного типа `int` и фиксированного `k` на практике часто ведет себя близко к линейному по `n`

ВАЖНО:
Этот вариант работает для НЕОТРИЦАТЕЛЬНЫХ int.
```cpp
void countingSortByDigit(int arr[], int n, int exp)
{
    // output - временный массив для стабильной сортировки по текущему разряду
    int* output = new int[n];
    int count[10] = { 0 }; // частоты цифр 0..9

    // 1) Считаем частоты цифр текущего разряда
    for (int i = 0; i < n; ++i)
        ++count[(arr[i] / exp) % 10];

    // 2) Превращаем count в префиксные суммы (позиции в output)
    for (int i = 1; i < 10; ++i)
        count[i] += count[i - 1];

    // 3) Заполняем output справа налево для стабильности
    for (int i = n - 1; i >= 0; --i)
    {
        int digit = (arr[i] / exp) % 10;
        output[count[digit] - 1] = arr[i];
        --count[digit];
    }

    // 4) Копируем результат обратно в arr
    for (int i = 0; i < n; ++i)
        arr[i] = output[i];

    delete[] output;
}

void radixSort(int arr[], int n)
{
    if (n <= 1)
        return;

    // Находим максимум, чтобы понять число проходов по разрядам
    int maxValue = arr[0];
    for (int i = 1; i < n; ++i)
    {
        if (arr[i] > maxValue)
            maxValue = arr[i];
    }

    // exp = 1 (единицы), 10 (десятки), 100 (сотни), ...
    for (int exp = 1; maxValue / exp > 0; exp *= 10)
        countingSortByDigit(arr, n, exp);
}
